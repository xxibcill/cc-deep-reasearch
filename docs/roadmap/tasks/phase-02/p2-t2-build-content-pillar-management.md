# P2-T2 - Build Content Pillar Management

## Summary

Make content pillars first-class strategy records that can be managed individually.

## Scope

- Add pillar list, add, edit, delete, reorder, and archive flows.
- Support structured pillar metadata such as summary, audience fit, proof notes, and platform preferences.
- Remove dependence on comma-separated pillar editing.

## Deliverables

- Pillar management UI components
- Pillar editing state and persistence wiring
- Tests for pillar CRUD and ordering flows

## Dependencies

- P1-T1 content pillar schema
- P2-T1 workspace structure

## Acceptance Criteria

- Users can manage multiple content pillars separately.
- Pillar updates persist correctly through the strategy API.
