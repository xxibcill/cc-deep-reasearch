# Task P2-T1: Backlog And Scoring Traceability

## Objective

Make backlog generation and idea scoring explicitly traceable to the opportunity brief.

## Scope

- Map ideas to audience, problem, sub-angle, and proof constraints.
- Add an opportunity-fit dimension to scoring.
- Preserve trace links in pipeline context or stage metadata.

## Acceptance Criteria

- Generated or shortlisted ideas can explain how they satisfy the brief.
- Scoring includes an explicit measure of opportunity fit.
- Operators can inspect the trace without reading raw prompts.

## Advice For The Smaller Coding Agent

- Add trace fields that are compact and stable, not prompt dumps.
- Keep the first scoring integration simple and explainable.
