# Phase 07 - Performance And Rule Updates

## Functional Feature Outcome

Performance learnings continuously update scoring, packaging, reuse, and time-budget rules so the workflow improves week over week.

## Why This Phase Exists

The current workflow has a feedback loop in theory, but not a tight enough one to affect throughput. This phase makes learning always-on, tracks operating fitness alongside content performance, and feeds updates back into earlier decisions without waiting for a full redesign cycle.

## Scope

- collect always-on performance, throughput, and reuse signals
- feed performance learnings back into scoring rubrics, hooks, packaging rules, and strategy constraints
- measure whether the new workflow is actually faster and more practical for a lean team

## Dependencies

- earlier phases must emit explicit decisions, timeboxes, and final publish states
- telemetry and performance analysis outputs should be extended rather than forked

## Exit Criteria

- performance outputs can change next-run rules without manual doc rewriting
- operators can see whether the workflow is getting faster, not just whether content performed well
- the team can measure kill rate, reuse rate, and cost control alongside audience outcomes
