# Phase 03 - Research And Argument

## Functional Feature Outcome

Research depth, proof requirements, and argument design scale to the upside and risk of the idea, and unsupported ideas can be stopped before drafting.

## Why This Phase Exists

The strongest part of the current workflow is the middle, but it is still too linear and too expensive by default. This phase keeps the rigor while introducing depth routing, earlier fact-risk checks, and a single thesis lane that can end in approval, hold, or kill instead of automatically flowing into scripting.

## Scope

- route research depth by ROI, novelty, and fact risk instead of using one default level
- merge angle choice and argument design into a single thesis and evidence lane
- start uncertainty and claim-risk review before draft creation

## Dependencies

- Phase 02 must produce explicit content type, effort tier, and decision metadata
- `ClaimLedger` and current research-pack outputs should be extended rather than duplicated

## Exit Criteria

- each selected idea exits with an approved thesis, a hold-for-proof decision, or a kill decision
- research depth and time budget are visible and enforced for every run
- claim status and uncertainty are available before draft generation starts
