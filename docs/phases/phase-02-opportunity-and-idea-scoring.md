# Phase 02 - Opportunity And Idea Scoring

## Functional Feature Outcome

The workflow can kill weak ideas early, branch by content type, and cap effort before research begins.

## Why This Phase Exists

The current backlog and scoring flow is complete, but it still lets low-value ideas consume too much process. This phase turns ideation into a decision lane with explicit ROI gates, effort controls, and content-type branching so the team does not run the full machine on every candidate.

## Scope

- merge opportunity planning, backlog generation, and scoring into a tighter evaluation lane
- add ROI, effort, and kill thresholds that determine whether an idea advances
- branch required depth by content type, channel intent, and publish goal

## Dependencies

- Phase 01 must define the operating contract and required governance fields
- existing backlog, scoring, and brief models should remain backward-compatible where practical

## Exit Criteria

- every candidate idea receives a produce, hold, reuse, or kill decision before research starts
- selected ideas carry a content type, effort tier, and expected upside
- low-value ideas cannot enter the research lane without an explicit operator override
